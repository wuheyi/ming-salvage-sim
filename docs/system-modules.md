# 系统模块索引

> **回合口径**：游戏主循环以“回合”为推进单位。**当前一回合 = 一个月**（语义可后续切换，代码里时段字段叫 `period`）。一年 12 回合，1627 年末到 1644 年约 200 个回合。资源收入、财政核算、科技推进、地区变化、军队补给和人物状态默认按回合结算；重大事件可在回合内切入临时处理。

回合流程：

1. 回合初奏报：事件、财政、军情、灾荒、地方、人物、党派、科技项目汇总。
2. 召对与廷议：玩家通过 LLM 对话问政、试探、任命、追责、下旨。
3. 旨意生成：自然语言转成结构化命令，挂接到对应模块。
4. 执行结算：人物、官职、党派、人群、地区、军队、物资共同影响结果。
5. 财政核算：国库和内库逐项登记收入、支出、转账、亏空和截留。
6. 月末事件：根据数值阈值触发新事件、隐患、回奏、弹劾、战报或民变。

## 模块关系

- 事件模块提出问题，旨意模块把皇帝对话转成行动。
- 经济模块为所有行动提供银粮，并记录每一笔流向。
- 人物通过官职获得执行权，通过党派和人际关系改变执行意愿。
- 地区承载人群、建筑、物资、军队和税收。
- 人群状态决定民心、动乱、税基、兵源和事件触发。
- 军队消耗国库、粮食和物资，也决定边防压力与地方安全。
- 科技需要人物、建筑、物资和资金，成功后反过来改变经济、军队和地区。
- 党派影响人物、官职、地区和舆论，是阳奉阴违的主要来源。
