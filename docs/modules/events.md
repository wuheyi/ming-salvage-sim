# 事件模块

事件模块负责把天下问题送到皇帝面前。玩家不直接操作事件表，而是通过奏报、召对和圣旨处理事件。

## 核心数值

- `紧急度`：本月是否必须处理。
- `严重度`：不处理会造成多大损害。
- `可信度`：奏报有多少可能是真话。
- `牵涉利益`：触动哪些人物、党派、地区或军队。

## 事件类型

- 朝政：弹劾、任免、清算、党争、制度争议。
- 财政：亏空、催饷、税收不足、借款、抄没。
- 军事：边警、战役、欠饷、哗变、将领争功。
- 地方：灾荒、粮价、逃户、盗匪、士绅请愿。
- 人物：求见、密奏、请辞、背叛、病亡。
- 科技：试验、事故、保守派攻击、推广受阻。

## 回合使用

每月给玩家 2-5 条关键奏报。玩家可以召见相关大臣追问细节，也可以直接下旨。事件结算后只告诉玩家“结果、代价、隐患”，不暴露完整后台公式。
