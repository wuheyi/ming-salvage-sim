# 开发路线草案

## 阶段 1：LLM 文字原型

目标：验证“崇祯重生 + 君臣对话 + 旨意执行结算”是否好玩。

- 做一个纯文本回合原型。
- 时间范围先压到 1627-1630。
- 每月给玩家一次奏报汇总。
- 每月允许安排若干场关键召对、廷议或密召。
- 玩家用自然语言问话、试探、任命、追责和下旨。
- LLM 扮演大臣、太监、将领、地方官和密探。
- 系统把玩家旨意解析成结构化对象。
- 用隐藏执行评估 Agent 判断执行结果、阳奉阴违、瞒报和事件后果，代码只负责校验与入库。
- 月末统一结算财政、科技、地区、军队、人物和党派变化。

## 阶段 2：政务与数值面板

目标：让玩家读懂“朕说了什么”和“天下实际做了什么”之间的差距。

- 做全国局势面板。
- 做朝堂、财政、军事、民生、内廷、人物 6 个页签。
- 显示国库、内库、民心、皇威、边防、民变、党争、官僚执行力、瞒报风险。
- 显示每道旨意的执行者、资源、期限、公开度、风险和回奏。
- 用事件日志记录“旨意原文 -> 结构化旨意 -> 执行变形 -> 后果”。

## 阶段 3：剧本扩展

目标：让每局出现不同历史分歧。

- 扩展到 1644 年完整主线。
- 加入两京十三省、辽东、东江镇、宣大蓟镇和漕运线。
- 加入袁崇焕、洪承畴、孙传庭、卢象升、李自成、皇太极等关键人物。
- 加入东林、阉党残余、宗室、边镇、士绅、商帮等内部团体。
- 加入南迁、议和、军政托管、中兴和保存火种等路线。

## 第一版数据表建议

- `events`：紧急度、严重度、可信度、牵涉利益。
- `economy`：国库、内库、月度收入、月度支出、欠账。
- `technologies`：皇帝认知点、超时代等级、项目进度、资源需求、反弹风险。
- `characters`：忠诚、能力、清廉、胆略、派系。
- `factions`：影响力、亲疏、利益受损、反弹风险。
- `population_groups`：规模、民心、负担、组织度。
- `regions`：民心、动乱、税收、田地、人群。
- `armies`：兵力、士气、训练、装备、欠饷。
- `offices`：权力、责任、声望、腐败空间。
- `buildings`：等级、完好、产出、维护、风险。
- `materials`：数量、质量、位置、价格。
- `imperial_metrics`：国库、内库、民心、皇威等全局核心数值。
- `pressures`：边防、民变、党争、灾荒、瞒报等全局压力。
- `edicts`：玩家自然语言旨意解析后的结构化命令。
- `execution_results`：旨意执行、扭曲、拖延、阳奉阴违和反噬记录。
- `audiences`：召对场景、可出席人物、对话目标和风险。
- `endings`：结局条件与叙事文本。

详细模块草案见 [system-modules.md](system-modules.md)。

## 第一版验证问题

- 玩家是否能通过自由对话获得有用信息，而不是只看固定选项？
- 同一道旨意用不同话术、权限和执行者时，结果是否明显不同？
- 阳奉阴违是否能被玩家从奏报和后果中推理出来？
- 国库、内库、民心、皇威是否形成有压力的取舍？
- 大臣是否像有利益和性格的人，而不是政策按钮？
