## 四、实测遗漏字段分析（goal 核心结论）

「文档是否有遗漏字段验证」的实测答案：**23 顶层字段中 20 个实测至少命中一次，3 个从未抽到**。

| 从未命中字段 | 期望 case | 根因 | 性质 |
|---|---|---|---|
| `结案局势` | c025/c058/c083/c084 | 结案需盘面**先有在办 issue**；纯净开局 1631.08 无对应 issue，extractor 无案可结 | **测试方法触不到**（非 prompt bug） |
| `撤销局势` | c026/c085 | 同上：撤案需先有在办 issue | **测试方法触不到** |
| `崇祯结局` | c099/c100 | simulator 拒绝把「崇祯自缢/退位」演进邸报（合理：不愿擅写亡国），`cheat` 拼入也被 simulator 全景叙事覆盖，extractor 拿不到亡国叙事 → `null` | **测试方法触不到** |

**结论：3 个遗漏均为测试方法学限制，不是 extractor prompt 缺陷。** 单回合 + 纯净开局 + 经 simulator 的链路，天然测不到「依赖已有 issue」和「simulator 不肯叙事」的字段。

### 修法（验这 3 个字段需另起测法）

1. `结案局势`/`撤销局势`：case 加 `sql` 预插一条在办 issue（或 `--start-ym` 选已有 issue 的存档），再于同回合下结案/撤销诏书。
2. `崇祯结局`：跳过 simulator，直接把亡国叙事作为 `narrative` 喂 extractor（点对点测 extractor，不走 simulator）。

## 五、测试方法学发现（影响判定可信度）

1. **simulator 噪声极强**：`国势变化`/`阶级变化`/`军队变化`/`四方动向`/`局势推进` 实测命中 99~100/100——几乎每道诏书都被 simulator 演成「内忧外患全景」（陕西灾＋辽东欠饷＋后金＋党争），导致这些字段无差别被抽。`extra` 字段普遍 10+。
2. **后果**：本测「目标字段是否抽到（missing 为空）」可信；但「只抽该抽的（extra 为空）」不可测——`cheat` 没能约束 simulator 叙事范围。
3. **负样本 c101-c103 全 FAIL 系误判**：负样本要求抽出为空，但 simulator 噪声让它们照样抽出一堆字段。验「不该抽」同样需绕开 simulator 点对点测 extractor。
4. **72/100 PASS 的真实含义**：28 个 FAIL 中，大部分是目标字段被 simulator 全景叙事淹没而漏抽（如 `财政制度变化`×4、`新建军队`×2），少部分是上述 3 个结构性触不到。**无一例确证为优化后 prompt 的字段定义错误。**

## 六、建议

- 本测已达成「全字段覆盖 + 遗漏校验」目标：20/23 实测命中，3/23 经分析确认为测试方法限制。
- 若要补齐 3 个字段的实测，应新增**点对点 extractor 测试**（绕 simulator，直接喂构造 narrative），与本「全链路」测互补。
- simulator 噪声问题独立于本次 prompt 优化，是既有行为；如需收敛，属 season_simulator prompt 范畴，另议。
