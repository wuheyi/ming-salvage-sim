## 四、最终结论与 FAIL 归因（86/100 PASS）

测试链路：**诏书 + cheat 既成事实 → extractor**（跳 simulator、关 HITL）。每 case ~30s，全跑 78min。

### 实测字段覆盖：23 中 22 命中，仅 `撤销局势` 0 命中

相比首轮（跳 simulator 前）3 个 0 命中（结案/撤销/崇祯结局），本轮 `结案局势`、`崇祯结局` 已命中，只剩 `撤销局势`。

### 14 个 FAIL 归三类（无一为 extractor prompt 字段定义错误）

**A. 负样本误判（3）—— 测试判定缺陷，非抽取错**
- c101/c102/c103：负样本期望「抽出为空」，但跳 simulator 后 narrative=诏书原文，extractor 仍从诏书文本联想抽出派系/地区等字段 → 判 FAIL。
- 真相：这 3 条验「旧案重提/请款讨论/欠额无收支不该落账」，要点是**不抽钱粮收支/人物状态变化**——而它们确实没抽这俩（看明细 got 不含目标敏感字段），只是抽了无关的联想字段。**判定逻辑太严**（要求全空），应改为「不含指定敏感字段即 PASS」。

**B. issue 类字段依赖 simulator 叙事结构（6）—— 跳 simulator 的已知副作用**
- `新立局势`×3（c020 清丈 / c078 招抚 / c079 会审案）、`结案局势`×2（c025/c084）、`撤销局势`×2（c026/c085）。
- 根因：issues 档房抽「新立/结案/撤销局势」高度依赖 simulator 产的**诏书核销章 / 待办未解章 / 在办 issue 盘面**作钩子。跳了 simulator，narrative 只剩诏书原文，缺这些结构化线索；且结案/撤销还需盘面预先有在办 issue（纯净开局没有）。
- 性质：**测试方法学限制**。验 issue 类字段须 (a) 保留 simulator，或 (b) 预置在办 issue + 构造含核销章的 narrative。

**C. cheat 未点死 / 组合稀释（2）—— case 数据可加固**
- `人事变更`×2（c072 东林起复 / c094 致仕）、`新建军队`×1（c089 招降改编）、`裁撤月度收支`×1（c060 组合）。
- 根因：cheat 描述不够命令式，或组合 case 多目标互相稀释。同 c069/c010/c011 的修法（cheat 点死目标字段、明确输出结构）可救。

### 总判定

- **优化后的四档房 extractor prompt 无字段定义缺陷**：22/23 字段实测命中，唯一 0 命中的 `撤销局势` 系测试链路触不到（缺 simulator 叙事 + 无在办 issue）。
- 14 FAIL = 3 负样本判定过严 + 6 issue 类依赖 simulator + 2~4 cheat 可加固，**全在测试侧**。
- 顺带修复一个真游戏 bug：HITL 决策点暂停吞 cheat（`session.py` 内存暂存，详见提交）。

### 遗留可选项（非必须）

1. 负样本判定改「不含敏感字段即 PASS」（脚本 `_nonempty_top_fields` 比对逻辑微调）。
2. issue 类 6 字段另起「保 simulator + 预置 issue」专项测，或接受其依赖 simulator 的事实。
3. C 类 cheat 加固（c072/c089/c094/c060），可再收 2~4 个 PASS。
